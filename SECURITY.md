# Security Policy

We are committed to keeping the portfolio repository secure. Please read the policy below to understand version support and how to report vulnerabilities safely.

## Supported Versions

Security updates are focused on the latest maintained line of the main branch.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue. Public issues disclose vulnerability details before a fix is available.

Instead, please use the GitHub Private Vulnerability Reporting feature for this repository:

1. On GitHub, navigate to the main page of the repository.
2. Under the repository name, click Security.
3. In the left sidebar, click Advisory.
4. Click Report a vulnerability.

If the private vulnerability reporting feature is not available, please open a minimal public Issue with a brief statement requesting a private contact path. Do not include exploit details, logs, credentials, or sensitive context in the public Issue description.

## Sanitize Your Submissions

Always sanitize your reports:
- Never disclose real API keys, passwords, credentials, tokens, or private environment settings.
- Ensure all logs, screenshots, and configuration snippets are fully redacted before submission.
